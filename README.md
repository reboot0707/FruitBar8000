# FruitBar8000 音樂查詢儀表板
- 命名: 期許成為下一個 Foobar2000
- 今天範疇: 資料表基礎設計、基礎關鍵 Metadata 的增刪查改

## 功能特色
- 支援多對多關聯式資料表 (見 XX 節)
- 支援「軟刪除」，以維資料安全
- 類「三層式架構維護」：
    1. 資料層: 統一由 EDMX (EntityFramework 6.x) 維護, 實現物件化資料管理, 系統運行時沒事(沒特殊需求)不要手刻 SQL.
    2. 業務邏輯層: 抽出「增」(Create)、「查」(Read)、「改」(Update)、「刪」(Delete)邏輯，適當抽取與 UI 無關的操作，增加可維護性，期許降低抽一根積木就全倒的風險。
    3. UI層: 以 WinForm 實作, 利用基本事件、委派觀念將互動邏輯精簡化、可擴充化。
